01. 변수 : 데이터 저장

변수(Variable): 변수는 값을 저장하기 위한 기억 공간을 의미합니다. JavaScript에서 변수는 var, let, const 등의 키워드를 사용하여 선언될 수 있습니다. 변수는 데이터를 저장하고 이를 참조하거나 조작하는데 사용됩니다.
데이터 불러오기(Data Retrieval): 데이터 불러오기는 일반적으로 외부 소스에서 데이터를 가져오는 작업을 의미합니다. 이것은 파일 시스템, 데이터베이스, 웹 API, 또는 사용자의 입력 등 다양한 소스에서 데이터를 가져올 수 있습니다. 

{
let x = 100, y = 200, z = "javascript";

console.log(x, y, z);
}

*변수 선언 및 초기화* 1. let: let 키워드는 블록 스코프(block scope)를 가지는 지역 변수를 선언할 때 사용됩니다. let으로 선언된 변수는 선언된 블록, 문장 또는 표현식 내에서 유효하며, 이를 벗어나면 해당 변수는 접근할 수 없습니다. 또한, let으로 선언된 변수는 재할당이 가능하다는 특징을 가집니다. 2. x = 100, y = 200, z = "javascript";: 이 부분에서는 세 개의 변수 x, y, z를 선언하고 각각의 초기값으로 100, 200, "javascript"를 할당하고 있습니다. 변수에 값을 할당할 때는 = 연산자를 사용합니다.

02. 상수 : 데이터 불러오기

데이터를 상수에 할당하여 사용: 외부 소스에서 가져온 데이터를 자바스크립트의 상수에 할당하여 사용하는 것을 의미할 수 있습니다. 이 경우에는 한 번 할당된 값은 변경되지 않으므로 상수를 통해 데이터를 안정적으로 저장할 수 있습니다.
데이터를 불러와서 상수에 할당: 외부 소스에서 데이터를 가져와서 자바스크립트의 상수에 할당하는 작업을 의미할 수 있습니다. 이 경우에는 데이터를 변경할 필요가 없으므로 상수를 사용하여 데이터를 안전하게 보관할 수 있습니다.
이 예시에서는 const 키워드를 사용하여 상수를 선언하고, 이들 상수에 각각 숫자와 문자열 값을 할당한 후, 이들 값을 콘솔에 출력하는 과정을 보여주고 있습니다. 상수는 프로그램 실행 중 값을 변경할 필요가 없는 데이터를 저장하는 데 유용하게 사용됩니다. 

{
const x = 100, y = 200, z = "javascript";

console.log("02. 상수 : 데이터불러오기");
console.log(x, y, z);
}

const 키워드는 상수를 선언할 때 사용됩니다. 상수는 선언과 동시에 초기화(값을 할당)되어야 하며, 그 이후에는 그 값이 변경될 수 없습니다. 만약 상수의 값을 변경하려고 시도하면, JavaScript는 오류를 발생시킵니다. 상수는 한 번 할당된 값이 변경되지 않는 변수를 의미합니다.

03. 배열 : 데이터 불러오기

"자바스크립트 배열의 데이터 불러오기"는 자바스크립트에서 배열을 사용하여 데이터를 가져오는 작업을 의미합니다. 배열은 여러 개의 값을 하나의 변수에 저장하는 자료구조로, 인덱스(index)를 사용하여 각 요소에 접근할 수 있습니다.
배열 생성: 데이터를 담을 배열을 먼저 생성해야 합니다. 배열은 대괄호 [] 안에 쉼표로 구분된 요소들로 정의됩니다.
데이터 불러오기: 배열에 저장된 데이터를 불러오려면 배열의 인덱스를 사용하여 해당 요소에 접근합니다. 배열의 인덱스는 0부터 시작하며, 각 요소에 대해 인덱스를 지정하여 해당 요소의 값을 얻을 수 있습니다.
배열을 사용하여 데이터를 불러오는 것은 자바스크립트에서 매우 일반적이고 유용한 작업입니다. 배열은 여러 가지 데이터를 하나의 변수에 효율적으로 저장하고 다룰 수 있도록 해주는 중요한 자료구조입니다. 

{
const arr = [100, 200, "javascript"];

console.log("03. 배열 : 데이터 불러오기");
console.log(arr[0], arr[1], arr[2]);
}

const 키워드를 사용하여 arr이라는 이름의 상수 배열을 선언합니다. 상수 배열의 경우, 배열 자체를 재할당하는 것은 불가능하지만, 배열 내의 요소를 변경하는 것은 가능합니다. 여기서는 배열을 재할당하지 않고, 단지 배열 내부의 데이터를 불러오기만 하므로 const를 사용하는 것이 적절합니다.
두 번째 console.log(arr[0], arr[1], arr[2]);는 arr 배열의 첫 번째 요소(arr[0]), 두 번째 요소(arr[1]), 세 번째 요소(arr[2])를 콘솔에 출력합니다. 배열의 인덱스는 0부터 시작하므로, arr[0]은 배열의 첫 번째 요소, arr[1]은 두 번째 요소, arr[2]는 세 번째 요소를 의미합니다.

04. 배열 : 데이터 불러오기 :갯수

자바스크립트에서 배열의 데이터 개수를 불러오는 것은 배열 내에 저장된 요소의 총 수를 확인하는 과정입니다. 이는 배열을 다룰 때 매우 기본적이면서도 중요한 작업 중 하나로, 배열의 길이를 알아내는 데 사용됩니다. 배열의 length 속성을 사용하면 배열에 저장된 요소의 총 개수를 쉽게 알 수 있습니다.
배열에서 데이터를 불러오면서 데이터의 갯수를 파악하는 작업을 의미합니다.
여기서는 배열의 요소를 하나씩 순회하면서 데이터를 불러오면서 동시에 배열의 요소의 갯수를 파악하는 방법에 대해 설명하겠습니다.
배열 생성: 먼저 배열을 생성하여 데이터를 저장합니다. 배열은 대괄호 [] 안에 쉼표로 구분된 요소들로 정의됩니다.
데이터 불러오기 및 갯수 확인: 배열의 length 속성을 사용하여 배열의 요소 갯수를 확인할 수 있습니다. 또한 반복문을 사용하여 배열의 요소를 하나씩 순회하면서 데이터를 불러오면서 동시에 갯수를 파악할 수 있습니다. 

{
const arr = [100, 200, "javascript"];
console.log(arr.length);    //배열 안에 객체의 갯수가 총 몇개 들어있는지 확인 하는 방법
}

JavaScript에서 배열의 길이(즉, 배열이 포함하고 있는 요소의 총 개수)를 확인하려면 .length 속성을 사용합니다. 예를 들어, arr.length는 배열 arr의 길이를 반환합니다. 아래 코드에서 arr.length는 3을 반환합니다. 이는 배열 arr이 세 개의 요소(100, 200, "javascript")를 포함하고 있기 때문입니다.

05. 배열 : 데이터 불러오기 :for()문

for문은 프로그래밍에서 반복적인 작업을 수행할 때 사용되는 가장 기본적인 제어문 중 하나입니다. 특정 조건이 만족될 때까지 코드 블록을 반복 실행하게 해주며, 주로 배열이나 컬렉션의 모든 요소를 순회할 때 사용됩니다.
아래는 배열의 데이터를 불러오는 과정에 대한 설명입니다.
배열 생성: 먼저 배열을 생성하여 데이터를 저장합니다. 배열은 대괄호 [] 안에 쉼표로 구분된 요소들로 정의됩니다.
자바스크립트에서 배열의 데이터를 불러오는 작업을 위해 for 반복문을 사용할 수 있습니다. for 반복문은 배열의 각 요소를 순회하면서 원하는 작업을 수행하는 데에 사용됩니다. 아래는 배열의 데이터를 불러오는 과정에 대한 설명입니다.
배열 생성: 먼저 배열을 생성하여 데이터를 저장합니다. 배열은 대괄호 [] 안에 쉼표로 구분된 요소들로 정의됩니다. for 반복문 사용: for 반복문을 사용하여 배열의 각 요소를 순회합니다. 이를 통해 각 요소에 접근하고 원하는 작업을 수행할 수 있습니다.

for (let i = 0; i < arr.length; i++) {       //초깃값; 조건식; 증감;
    console.log(arr[i]);
}
}

06. 배열 : 데이터 불러오기 :forEach()

forEach 메서드는 배열을 순회하는 데 사용되는 메서드 중 하나입니다. 이 메서드는 배열의 각 요소에 대해 주어진 함수를 한 번씩 실행합니다. forEach는 배열의 요소들을 반복하여 작업을 수행할 때 유용하며, 함수형 프로그래밍 패러다임을 따르는 코드를 작성할 때 자주 사용됩니다.
*forEach() 메소드는 배열의 각 요소에 대해 주어진 함수를 실행하는 메소드입니다. 이 메소드는 배열 내의 각 요소를 순회하면서 콜백 함수를 한 번씩 실행합니다.콜백 함수는 최대 세 개의 인자를 받을 수 있으며, 이는 각각 현재 요소의 값, 현재 요소의 인덱스, 그리고 순회되는 배열 자체입니다.
*forEach() 메소드는 배열을 순회하며 각 요소에 대해 특정 작업을 수행할 때 유용하게 사용됩니다. 그러나 이 메소드는 순회를 중간에 멈출 수 없다는 특징이 있습니다(즉, break나 return을 사용할 수 없습니다). 필요한 경우, 다른 반복문이나 배열 메소드(for, for...of, map, filter 등)를 사용할 수 있습니다. 

{
const arr = [100, 200, 300, 400, 500, 600, 700, 800, 900];

//01

})
//02
arr.forEach(function (element, index) {
    console.log(index);
})
//03
arr.forEach(function (element, index, array) {
    console.log(array);
})
}

07. 배열 : 데이터 불러오기 :for of()

for...of 문은 반복 가능한 객체(Array, Map, Set, String 등)를 순회하며 그 값을 변수에 할당합니다. 배열의 경우, 각 순회에서 배열의 요소 값이 변수에 할당됩니다.
이 코드에서 for of 문은 배열 arr의 각 요소를 순회하면서, 변수 element에 배열의 요소 값을 할당하고, 그 값을 콘솔에 출력합니다. 두 코드 예제는 결과적으로 같은 값을 출력하지만, 이들의 사용 목적과 적용 범위가 다르므로, 상황에 맞게 적절한 반복문을 선택하여 사용하는 것이 중요합니다. 

{
const arr = [100, 200, 300, 400, 500, 600, 700, 800, 900];

for (let element of arr) {
    console.log(element);
}
}

08. 배열 : 데이터 불러오기 :for in()

for in 문은 객체의 모든 열거 가능한 속성을 순회하며, 각 속성의 키(또는 인덱스)를 변수에 할당합니다. 배열을 순회할 때, 인덱스가 변수에 할당되며, 객체를 순회할 때는 키가 할당됩니다.
이 코드에서 for in 문은 배열 arr의 인덱스를 순회하면서, 변수 element에 인덱스를 할당하고, arr[element]를 통해 해당 인덱스의 배열 요소 값을 콘솔에 출력합니다. arr안에 배열 선언 없이 element만 입력시 키값만 불러옵니다. ex : (0 1 2 3 ...). 

{
const arr = [100, 200, 300, 400, 500, 600, 700, 800, 900];

for (let element in arr) {
    console.log(arr[element]);
}
}

for in 문과의 차이점
*목적과 사용법: for of는 반복 가능한 객체의 값을 직접 순회하기 위한 목적으로 사용되며, for...in은 객체의 모든 열거 가능한 속성(키 또는 인덱스)을 순회하기 위해 사용됩니다.
*배열 사용 시: for of는 배열의 각 요소의 값에 접근하는 데 사용되며, for in은 배열의 인덱스를 순회합니다. for in을 배열에 사용할 경우, 배열의 인덱스가 문자열로 처리되며, 배열 이외의 열거 가능한 속성도 순회할 수 있기 때문에 예상치 못한 결과를 초래할 수 있습니다. 따라서 배열을 순회할 때는 주로 for of를 사용하는 것이 좋습니다.
*객체 사용 시: for in은 객체의 속성을 순회할 때 주로 사용되며, 객체의 키(속성 이름)에 접근하기에 적합합니다. for of는 직접 객체에 사용할 수 없고, 객체가 반복 가능한 프로토콜을 구현한 경우(예: Map, Set)에만 사용 가능합니다.

09. 배열: 데이터 불러오기: map()

map() 메서드는 배열의 모든 요소에 대해 주어진 함수를 호출하고, 결과를 모아 새 배열로 반환합니다. map()은 호출된 배열을 변형시키지 않으며, 원본 배열과 동일한 길이의 새 배열을 생성합니다. map()의 기본 구조는 다음과 같습니다 element: 현재 처리하고 있는 요소의 값입니다. index: 현재 처리하고 있는 요소의 인덱스입니다. array: map() 메서드가 호출된 원 배열입니다.
*forEach() 메서드와의 차이*
forEach() 메서드도 배열의 각 요소에 대해 주어진 함수를 실행하지만, map()과 다르게 아무것도 반환하지 않습니다(undefined를 반환). 즉, forEach()는 순수하게 부작용(side effect)을 목적으로 하는 경우에 사용됩니다(예: 각 요소를 출력). 반면, map()은 배열의 각 요소를 변형하여 새로운 배열을 만들고자 할 때 사용됩니다. - 결론
*map(): 배열의 각 요소를 변형하여 새로운 배열을 만들고자 할 때 사용. 반환값은 변형된 요소를 포함하는 새 배열입니다.
*forEach(): 배열의 각 요소에 대해 주어진 함수를 실행할 때 사용. 반환값은 없습니다(즉, undefined). 

{
const arr = [100, 200, 300, 400, 500, 600, 700, 800, 900];

//01

    console.log(element)
});    //[100~900]  배열안에 감싸져있는 상태로 나옴
arr.map(function (element, index) {
    console.log(index)
});
arr.map(function (element, index, array) {
    console.log(array)
});
}

    console.log(array);

const arr = num.filter(function (element) {

11. 배열 : 데이터 불러오기 :배열 구조 분해 할당

배열 구조 분해 할당(Array Destructuring Assignment)은 JavaScript에서 배열을 해체하여 각 요소를 변수에 할당하는 방법입니다. 이를 통해 배열의 각 요소에 쉽게 접근할 수 있습니다.
이처럼 배열 구조 분해 할당을 사용하면 배열의 요소에 쉽게 접근하여 사용할 수 있습니다. 이 기능은 코드를 간결하게 작성할 수 있도록 도와주며, 특히 함수의 반환 값이 배열일 때 유용하게 사용됩니다. 

{
                    {
                        const arr = [100, 200, "javascript"];
                        const [a, b, c] = arr;
            
                        console.log(a);     //100
                        console.log(b);     //200
                        console.log(c);     //javascript
            
                    }
                    //선언에서 분리한 할당
                    {
                        const [a, b, c] = [100, 200, "javascript"];
            
                        console.log(a);     //100
                        console.log(b);     //200
                        console.log(c);     //javascript
            
                    }
                    //기본 값 설정
                    {
                        const [a, b, c = "javascript"] = [100, 200];
            
                        console.log(a);     //100
                        console.log(b);     //200
                        console.log(c);     //javascript
            
                    }
                    {
                        const [a, , c = "javascript"] = [100, 200, "javascript"];
            
                        console.log(a);     //100
                        console.log(c);     //javascript
            
                    }
                    //나머지 값 할당
                    {
                        const [a, b, ...rest] = [100, 200, "javascript", "jquery"];
            
                        console.log(a);           //100
                        console.log(b);           //200
                        console.log(...rest);     //javascript jquery
                    }
                    {
                        //변수 값 교환
                        let a = 100, b = 200;
                        [a, b] = [b, a];
            
                        console.log(a);
                        console.log(b);
                    }
                    {
                        //다차원 배열 펼침연산자
                        const [a, [b, c]] = [1, [2, "javascript"]];
            
                        console.log(a);
                        console.log(b);
                        console.log(c);
                    }
}

이렇게 하면 객체의 속성에 간편하게 접근할 수 있으며, 코드를 보다 간결하게 작성할 수 있습니다. 객체의 속성 이름을 일일이 지정하는 대신 구조 분해 할당을 사용하여 속성에 접근할 수 있으므로 코드의 가독성이 높아집니다. 

{

        a: 100,

{
